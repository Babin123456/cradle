# Markdown Resume Generator Architecture

## Overview

Markdown Resume Generator is a static browser mini project that turns Markdown into a live, print-ready resume preview. It gives users a focused editor, template selection, theme selection, autosaved drafts, and export controls without requiring a backend or build step. The project is intentionally dependency-light so contributors can understand and modify it quickly.

---

## Purpose & Goals

- Generate a professional resume directly from Markdown.
- Provide live preview feedback while the user edits the resume source.
- Offer multiple resume templates and visual themes.
- Support practical export flows: copy Markdown, download Markdown, export HTML, and print/save as PDF.
- Keep the implementation self-contained and consistent with Cradle's mini-project structure.

## System / Project Architecture Overview

The project follows the standard Cradle static mini-project pattern: HTML defines the page structure, CSS owns presentation, and JavaScript owns behavior. The user edits Markdown in a textarea, `script.js` parses the supported Markdown subset into safe HTML, and the preview pane updates immediately with the selected template and theme classes.

There is no server, bundler, or npm dependency required for this mini project. Browser APIs handle local draft persistence, clipboard copy, file downloads, and print/PDF export.

```text
User opens index.html
        ↓
Browser loads style.css, Cradle UI helpers, and script.js
        ↓
Saved Markdown, template, and theme load from localStorage
        ↓
Event listeners are attached to editor and toolbar controls
        ↓
Markdown is parsed into escaped, resume-friendly HTML
        ↓
Preview re-renders with selected template and theme classes
        ↓
User copies, downloads, exports HTML, or prints/saves as PDF
```

## Data Flow / Execution Flow

```text
DOMContentLoaded fires
        ↓
initializeResumeGenerator() reads localStorage or falls back to DEFAULT_MARKDOWN
        ↓
Template and theme selectors are restored
        ↓
updatePreview() parses Markdown and applies preview classes
        ↓
User edits Markdown or changes template/theme
        ↓
updatePreview() runs again, updates word count, and persists state
        ↓
Export actions read the current editor/preview state
```

## Key Features

- Live Markdown-to-resume preview.
- Three templates: Classic, Modern, and Compact.
- Four themes: Slate, Indigo, Emerald, and Rose.
- Sample resume reset for quick testing.
- Copy Markdown to clipboard.

## File Responsibilities

### `index.html`

- Loads Google Fonts, Cradle UI tokens, local styles, Button helper, BackToHome helper, and project script.
- Defines the page header and short project description.
- Provides template and theme selectors.
- Provides export buttons for sample reset, copy Markdown, Markdown download, HTML export, and PDF export.
- Contains `#markdownInput` for source editing and `#resumePreview` for live output.

### `script.js`

- `DEFAULT_MARKDOWN` stores the sample resume used on first load and by the sample button.
- `initializeResumeGenerator()` restores saved editor state and wires all event listeners.
- `updatePreview()` updates template/theme classes, renders parsed Markdown, autosaves state, and updates word count.
- `parseMarkdown()` converts supported resume Markdown lines into HTML.
- `formatInline()` handles escaped inline formatting for bold, italic, and links.
- `copyMarkdown()` copies the current Markdown to the clipboard.
- `downloadMarkdown()` downloads the source as `resume.md`.
- `downloadResumeHtml()` exports a standalone `resume.html` file with embedded resume styles.
- `printResume()` calls `window.print()` for PDF export.
- `escapeHtml()` prevents raw user Markdown from being injected as HTML.

### `style.css`

- Defines the dark Cradle-style editor shell and responsive two-pane workspace.
- Styles the printable resume page separately from the editor interface.
- Provides `.template-classic`, `.template-modern`, and `.template-compact` layout variants.
- Provides `.theme-slate`, `.theme-indigo`, `.theme-emerald`, and `.theme-rose` accent variants.
- Uses `@media print` to hide controls and render only the resume preview for PDF output.

---

## Design Decisions

- **Native PDF export** — the project uses `window.print()` so users can save as PDF without a third-party dependency.
- **Small Markdown subset** — the parser supports resume-focused Markdown syntax instead of trying to implement the entire Markdown specification.
- **Escaped inline content** — Markdown text is escaped before formatting so user input is not treated as raw HTML.
- **Class-based templates and themes** — template and theme choices are CSS classes, making future visual variants easy to add.
- **Autosave through localStorage** — draft and preference persistence works offline and does not require a backend.
- **Cradle UI convention alignment** — the page loads shared Cradle tokens and helper scripts so the mini feels consistent with newer project pages.

## Known Limitations

- The parser does not support every Markdown feature.
- PDF export depends on browser print behavior and available print settings.
- The Modern template is conservative to keep print output stable.
- Draft data is stored only in the current browser through localStorage.

---

## Development Notes

- No build step is needed for the mini itself; open `index.html` or serve the repository with `python -m http.server 8000`.
- Run `npm run build` from the repository root after adding or renaming projects so `data/projects.json` and thumbnails stay updated.
- Run `node --check projects/editor/markdown-resume-generator/script.js` to verify JavaScript syntax.
- When changing supported Markdown syntax, update both `parseMarkdown()` and the README's supported Markdown section.
- Keep print-specific changes inside the `@media print` block so the editor UI remains unaffected.
